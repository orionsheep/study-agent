### 用户反馈（5 个问题）
1. 整个画面渲染不正常（dashboard + immersive）——液态玻璃变不透明、裂变图交互错位

### 根本诊断
**所有渲染问题的共同根因**：英语工作区被渲染在 `<div className="canvas-world" style={{ transform: scale() }}>` 内。CSS 规范下 transform 祖先创建合成层，导致后代 `backdrop-filter` 失效（液态玻璃变不透明）+ FissionGraph canvas 坐标错位（hover/click 失效）。

### 核心架构改动：NativeAppLayer
- 新增 `.native-app-layer`：canvas-world 的**同级**（canvas-viewport 内），workspace app 在这里渲染，脱离 transform
- `isWorkspaceAppType` 判断（english.workspace/humanities.notebook/english.dashboard）→ 从 canvas-world 的 visibleApps.map 过滤掉，改在 native-app-layer 渲染
- 用户选择"窗口尺寸缩放但内容保持原生" → native-app-layer 镜像 canvas-world 的 translate+scale，窗口跟随画板平移缩放，transform 完全同步验证通过

### FissionGraph 修复
- **dimensions 不更新根因**：ResizeObserver 在 react-resizable-panels Panel 异步布局下失效
- 修复：加 rAF + 多次 setTimeout + **polling 兜底**（200ms 间隔直到尺寸稳定）
- zoomToFit 时序修复（300ms 延迟让 force simulation 收敛）
- 修复后 canvas resolution = CSS 尺寸 = 容器尺寸，5 个节点可 hover + 点击切词验证通过
