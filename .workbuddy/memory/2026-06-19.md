## 英语工作区四项修复

1. **英语窗口始终置顶问题** — `styles.css` 的 `.canvas-world` 默认 z-index auto(0)，而 `native-app-layer` 非聚焦时 zIndex=5，导致英语工作区始终在 canvas-world 之上、遮挡其他窗口。修复：给 `.canvas-world` 显式设 `z-index: 10`，这样 native-app-layer(5) 非聚焦时沉到 canvas-world 之下，聚焦时跳到 60 浮到上方。

2. **星链图节点点不到** — `FissionGraph.tsx` 用 `getBoundingClientRect()` 测量容器尺寸，但该组件在 `native-app-layer` 里，该层带 CSS `transform: scale(viewport.scale)`，getBoundingClientRect 返回的是变换后（屏幕）像素而非 CSS 布局像素，导致 ForceGraph2D 画布尺寸与容器不匹配、点击命中区错位。修复：改用 `offsetWidth/offsetHeight`（始终返回 CSS 布局像素）。

3. **英语仪表盘整合到系统级仪表盘** — 把原 `EnglishDashboard`（英语模块内独立 stats 页）的词库/单词/掌握度数据整合进 `NativeAppRenderer.tsx` 的 `LearningDashboardApp`，系统级仪表盘（dashboard.learning）现在同时展示通用学习数据 + 英语学习统计。

4. **恢复英语工作区底部按钮栏** — `EnglishWorkspaceApp` 原来缺少底部功能入口（原 ThreeColumnLayout 有 Global Footer）。新增底部工具栏含「统计」「测验」「Immersive」按钮，并新增 quiz 模式接入 QuizPanel 组件（之前导入了但未使用）。

3. **SQLite → PostgreSQL 数据迁移** — 写了 `scripts/migrate_sqlite_to_postgres.py`，把 33 张表、87862 行全部迁入 PG。迁移中遇到并解决的类型适配问题：
   - `vector` 列（pgvector）：空/`[]` → NULL，非空字符串原样传入
   - `jsonb` 列：先 `json.loads` 再 `Json()` 包装（避免双重编码）
   - `text[]` 数组列（如 edu_memories.tags）：JSON 字符串 → Python list
   - `boolean` 列：0/1/smallint → Python bool
   - `timestamp` 列：`+00:00Z` → `+00:00`（PG 不接受时区偏移和 Z 同时存在）
   - 6 张 SQLite-only 表（artifacts/chat_resource_links/notebooks 等）自动在 PG 创建
   - 列差异（course_documents 多 8 列）自动 ALTER TABLE 补齐
   - 用 autocommit + DELETE（而非大事务 TRUNCATE）避免锁表卡死
